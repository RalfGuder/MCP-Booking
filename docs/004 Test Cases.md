## Test Case Struktur

Jeder Test Case folgt dieser Struktur:
- **Verknüpfungen** — Links zu UC, US und REQ
- **Testszenarien** — Nummerierte Szenarien (TS-xxx.yy)
  - Schicht und Typ (Application/Infrastructure/Server, Unit-/Integrationstest)
  - Vorbedingung, Aktion, Erwartetes Ergebnis
  - Status (Implementiert/Ausstehend) mit Referenz auf Testklasse/-methode
